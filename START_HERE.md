# Claude Code 첫 세션 시작 프롬프트

> 이 파일의 **하단 영역(```으로 둘러싸인 부분)을 통째로 복사**해
> Claude Code 첫 메시지로 붙여넣으세요. Claude Code가 `CLAUDE.md`를
> 읽고 컨텍스트를 잡은 뒤, 단계별로 작업을 시작합니다.

## 옵션 A: 데이터 검증부터 시작 (권장)

처음 받는 프로젝트면 먼저 데이터가 올바르게 들어왔는지 확인하는 것이 안전합니다.

```
@CLAUDE.md 를 먼저 읽고 이 프로젝트의 전체 컨텍스트를 파악해줘.

그 다음 단계별로 진행:

1단계: 데이터 무결성 검증
   - data/ 폴더의 3개 JSON 파일을 로딩
   - departments_dna.json: 31개 학과, 각 학과별 24개 매칭축 가중치가 모두 있는지
   - question_bank.json: 240문항, 1차/2차/정밀형이 90/50/100인지, ID 연속성
   - department_cards.json: 31개 학과 카드가 있는지
   - 검증 결과를 표로 보고

2단계: 적합도 엔진 회귀 테스트
   - node tests/test_engine.js 실행
   - 5명의 가상 학생 모두 의도한 학과가 TOP5에 들어가는지 확인
   - 통과 못 하면 lib/recommendation_engine.js를 점검

3단계: 작업 계획 제안
   - CLAUDE.md 12장에 정해진 작업 우선순위를 보고
   - 학생용 1차 기본검사 화면(src/student/index.html)을 어떻게 설계할지 제안
   - 기술 스택(Vanilla 단일 HTML vs Vite+React) 선택안 제시
   - 다음 단계 진행 전 내 승인을 받기

각 단계 완료 시 결과를 보고하고 다음 단계 진행 여부 확인.
```

## 옵션 B: 바로 학생용 1차 검사 화면부터 작성

데이터를 이미 신뢰하고 빠르게 프로토타입을 보고 싶다면:

```
@CLAUDE.md 를 읽고 전체 컨텍스트를 파악한 다음, 학생용 1차 기본검사 화면을
src/student/index.html 단일 파일로 만들어줘. 다음 조건을 지켜줘:

- 외부 의존성 없는 Vanilla JS + CSS (1차 시범운영용)
- 데이터는 fetch('../../data/question_bank.json')으로 로딩
- 1차 검사 90문항만 화면당 1문항씩 출력 (stage=1 필터)
- 5점 척도 버튼: "전혀 그렇지 않다 / 그렇지 않다 / 보통이다 / 그렇다 / 매우 그렇다"
- 진행률 바, 뒤로가기 가능, sessionStorage에 응답 누적
- 모바일 우선 디자인 (최소 320px 너비 대응)
- 90문항 완료 시 lib/recommendation_engine.js를 import해 매칭축 점수만 콘솔에 출력
  (결과지 화면은 다음 단계에서 작성)
- 한국어 UI, 학번/이름 입력 화면은 지금은 만들지 말고 닉네임만 받기 (개인정보 보호)
- 결과지 화면 안내문구는 CLAUDE.md 8장 참조

완성 후 어떻게 로컬에서 띄워볼 수 있는지(예: python -m http.server) 안내까지.
```

## 옵션 C: 적합도 산출 로직만 점검

코드 작성 전 적합도 계산이 계획서 공식과 정확히 일치하는지부터 보고 싶다면:

```
@CLAUDE.md 를 읽고, lib/recommendation_engine.js 의 적합도 산출 공식이
CLAUDE.md 3.5절의 공식과 정확히 일치하는지 검토해줘.

특히 다음을 확인:
1. 분자: Σ(학생 매칭축 점수 × 학과 DNA 가중치)가 24축 전체에 대해 계산되는지
2. 분모: 학과 DNA 가중치 합 × 5 가 학과별로 다르게 적용되는지
3. 역채점 처리가 calcAxisScores 에서 응답값 6-v 변환으로 되어 있는지
4. CONF/NEED 문항이 매칭축 계산에는 기여하지 않고 calcCounselingNeed 에만 쓰이는지
5. tests/test_engine.js 의 5명 가상 학생 시나리오를 실행해 TOP5 정확도 확인

이상 없으면 다음 단계로 학생용 1차 검사 화면 작성 제안.
이상 있으면 어디가 어떻게 다른지 보고하고 수정 방안을 물어보기.
```

## 어떤 옵션을 선택할까?

- **신중하게 진행하고 싶다** → **옵션 A**
- **빨리 결과물을 보고 싶다** → **옵션 B**
- **로직 정확성이 우선이다** → **옵션 C**

처음이라면 **옵션 A 권장**.

## Claude Code 사용 팁

- 매 세션 첫 메시지에 `@CLAUDE.md` 를 포함하면 가장 안전 (Claude Code가 자동으로 읽음)
- 작업 분기점마다 진행 보고 받기 (위 프롬프트에 명시되어 있음)
- 큰 변경(데이터 수정, 공식 변경)은 반드시 사용자 승인 후 진행 (CLAUDE.md 2장)
- 한 세션이 길어지면 CLAUDE.md를 갱신해 다음 세션 컨텍스트 보존
- 코드 변경 시 `git commit` 자주 (Claude Code가 알아서 제안함)
