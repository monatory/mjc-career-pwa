# src/admin/ — 관리자 대시보드 작업공간

학생용 PWA가 안정화된 뒤(=`src/student/` 완료 후) 작업할 영역입니다.

---

## 대시보드 구성 (계획서 Ⅹ장 그대로)

### 권한별 메뉴

| 권한 | 메뉴 |
|---|---|
| **AI융합진로지원센터** (최고권한) | 종합 현황, 학과별 추천 분포, 계열별 관심도, 학생별 결과(식별정보 포함), 상담 필요군 목록, 실제 선택 학과 입력, 만족도·상담결과 입력, 데이터 다운로드 |
| **융합교육지원센터** | 운영지표 요약 (개인정보 제거된 집계 데이터만) |
| **학과장** (해당 학과) | 학과 관심도 (본인 학과 추천·관심 추이, 인접 학과와의 경쟁 구도, 집계만) |

### 핵심 KPI

- **참여 추이**: 전체/일별/주별 참여자 수, 검사완료율, 평균 소요시간
- **학과별 추천 분포**: 31개 학과별 TOP1 비율, TOP5 누적 비율, 전년 대비 변화
- **상담 필요군**: 70점 이상 학생 자동 추출 → 상담 일정 배정 연계
- **추천 적중률 (Hit@5)**: 학기 종료 후 실제 선택 학과 입력 → TOP5 안에 포함된 비율 계산
- **만족도**: 1~5점 평균, 자유응답 코멘트 워드클라우드

---

## 보안·접근제어

- 학생 식별정보(학번·이름)는 AES-256 암호화 저장. 진로취업팀만 복호화 가능.
- 학과장 권한은 본인 학과의 **집계 데이터만** 조회 가능 (개별 학생 식별 불가).
- 데이터 다운로드 시 「개인정보 제거」 또는 「포함」 옵션 명시적으로 선택.
- 로그인은 학내 SSO 연동 권장 (직접 비밀번호 관리 지양).

---

## 권장 기술 스택

- **차트**: Recharts 또는 Chart.js (참여 추이 시계열, 학과별 분포 막대)
- **테이블**: 가벼운 데이터 그리드 (예: TanStack Table)
- **인증**: 학내 SSO (Microsoft Entra ID / Keycloak 등)
- **데이터 소스**: 학내 서버 API 또는 별도 백엔드 (현재 미정)

---

## 작업 시작 시점

`src/student/` 의 1차 기본검사 + 결과지가 완성되고 시범운영 데이터가
50~100명 누적된 후 본격 개발. 그 전에는 더미 데이터로 화면 와이어프레임만 작성.
