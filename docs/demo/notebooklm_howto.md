# NotebookLM으로 시연 영상 만들기 — 진행 순서

> 이 폴더의 대본(`*.md`)을 **NotebookLM**에 넣어 자동으로 슬라이드+한국어 내레이션
> 동영상(Video Overview)을 만드는 단계별 안내.
> ※ NotebookLM은 구글 서비스라 화면 메뉴 이름이 조금씩 바뀔 수 있습니다.
> 버튼 글자가 달라도 **흐름(소스 추가 → 언어 한국어 → Video Overview → 프롬프트 → 생성 → 다운로드)** 은 동일합니다.

---

## 0. 무엇을 보낼지 먼저 고르기

| 청중·용도 | 보낼 파일 | 길이 |
|---|---|---|
| 학생·학부모 홍보 (카톡·SNS) | `demo_1min.md` | 약 1분 |
| 일반 소개·검토 회의 도입 | `notebooklm_demo_3min.md` | 약 3분 |
| 학과장·교수진·운영위 심화 | `demo_5min.md` | 약 5분 |

**선택**: 위 대본 1개를 **메인 소스**로, [`CLAUDE.md`](../../CLAUDE.md)를 **보조 소스**로 함께 올리면
세부 수치·정책까지 정확히 인용됩니다. 화면 스크린샷([`assets/`](assets/))을 함께 올리면
영상 슬라이드에 실제 화면이 들어갑니다(선택).

## 1. NotebookLM 접속

1. 브라우저에서 **https://notebooklm.google.com** 접속
2. 구글 계정으로 로그인 (학교 구글 계정 권장)
3. **"새 노트북 만들기"**(Create new) 클릭

## 2. 소스 추가 (대본 보내기)

NotebookLM은 "소스(Sources)"에 올린 자료만 읽습니다.

1. 왼쪽 **"소스 추가"**(+ Add source) 클릭
2. 자료를 올리는 방법 — 둘 중 편한 것:
   - **(A) 파일 업로드**: 이 폴더의 `demo_3min.md`(또는 1min/5min) 파일을 선택
   - **(B) 텍스트 붙여넣기**: `.md` 업로드가 안 되면, 파일을 메모장으로 열어 전체 복사(Ctrl+A→Ctrl+C)
     → "복사된 텍스트"(Paste text)에 붙여넣기(Ctrl+V)
3. (권장) **CLAUDE.md도 같은 방식으로 추가** — 사실 정확도 ↑
4. (선택) `assets/01_step1_intro.png` 등 스크린샷도 소스로 업로드 → 영상 시각 자료로 활용
5. 올린 소스들의 **체크박스가 켜져 있는지** 확인 (영상은 켜진 소스만 사용)

> GitHub에서 바로 가져오려면: 각 파일 GitHub 페이지의 **Raw** 버튼 → 주소를 복사 →
> "소스 추가 → 링크(URL)"에 붙여넣어도 됩니다.
> 예: `https://github.com/monatory/mjc-career-pwa/blob/main/docs/demo/notebooklm_demo_3min.md`

## 3. 출력 언어를 한국어로

영상이 영어로 나오지 않게 미리 지정합니다.

1. 우측 상단 **설정(⚙️)** 또는 계정 메뉴 → **"출력 언어"**(Output language)
2. **한국어** 선택
3. (설정이 안 보이면 4단계 프롬프트에서 "한국어로"라고 명시하면 됩니다)

## 4. Video Overview(비디오 개요) 생성

1. 우측 **Studio**(스튜디오) 패널에서 **"Video Overview"**(비디오 개요) 찾기
   - 같은 패널에 "오디오 개요(팟캐스트)"도 있는데, 영상은 **Video** 쪽입니다
2. **"사용자 지정"**(Customize) 클릭 → 아래 프롬프트 붙여넣기:

   **3분판용 프롬프트 예시**
   ```
   한국어 내레이션, 약 3분. 학내 의사결정자와 학생·교수진이 함께 보는 혼합 청중용 시연.
   업로드한 대본의 SCENE 1~7 순서를 그대로 따르고, 각 SCENE의 화면 설명과 내레이션을
   슬라이드로 구성. 라이브 URL과 핵심 수치(8축·90문항·31학과·TOP5)를 슬라이드에 포함.
   ```

   **1분판용** → "약 1분, 학생 대상 홍보, SCENE 1~4, 친근한 톤"
   **5분판용** → "약 5분, 교수진·운영위 대상, SCENE 1~9, 산출 공식·거버넌스 강조"

3. **"생성"**(Generate) 클릭
4. 몇 분 기다립니다 (영상 길이·서버 상황에 따라 수 분 소요)

## 5. 확인하고 내려받기

1. 생성 완료되면 노트북 안에서 **재생**해 확인
2. 내용·발음·자막이 맞으면 **다운로드**(⋮ 메뉴 → Download) → **MP4** 저장
3. 어색하면 4단계 프롬프트를 다듬어 **다시 생성**

## 6. 활용

- **카톡·메일**: 저장한 MP4 첨부 (1분판 추천)
- **발표**: PowerPoint에 MP4 삽입 또는 OBS 발표 중 재생
- **추가 편집**: 자막·로고·BGM은 곰믹스·Clipchamp(윈도우 기본) 등에서 후가공

## 자주 막히는 곳 (체크리스트)

| 증상 | 해결 |
|---|---|
| 영상이 영어로 나옴 | 3단계 출력 언어 **한국어** 지정 후 재생성 (또는 프롬프트에 "한국어로") |
| 수치·학과명이 틀림 | `CLAUDE.md`를 소스로 추가하고, 프롬프트에 "업로드한 자료의 수치만 사용" 명시 |
| 영상이 너무 길거나 짧음 | 대본 자체를 1min/3min/5min 중 맞는 것으로 교체 + 프롬프트 길이 지정 |
| Video Overview 메뉴가 안 보임 | 계정·지역에 따라 순차 제공 중일 수 있음. 우선 **오디오 개요**로 대체하거나 며칠 후 재확인 |
| 무료 한도 초과 | NotebookLM 무료 플랜은 생성 횟수 제한이 있음. 다음날 재시도 또는 유료(Plus) |
| 화면 캡처가 안 들어감 | `assets/*.png`를 **소스로 업로드**했는지 확인. 텍스트만 있으면 NotebookLM이 자체 그래픽 생성 |

## 빠른 요약 (한눈에)

```
대본 고르기(1/3/5분)
  → notebooklm.google.com 새 노트북
  → 소스 추가: 대본.md (+ CLAUDE.md, +스크린샷)
  → 출력 언어 한국어
  → Studio → Video Overview → 사용자 지정(프롬프트) → 생성
  → 확인 → MP4 다운로드 → 카톡·발표 활용
```

대본 직접 녹화(NotebookLM 없이)는 [`cue_sheet.md`](cue_sheet.md) 참조.
