# 발표자 큐시트 — MJC-CAT 3분 라이브 시연

> OBS Studio / Windows 게임바(Win+G) / PowerPoint 화면녹화로 **직접 녹화**할 때 따라 읽는 진행표.
> NotebookLM 자동 영상([`notebooklm_demo_3min.md`](notebooklm_demo_3min.md))의 SCENE 1~7과 1:1 정합.
> 화면 레퍼런스는 [`assets/`](assets/) 스크린샷 팩.

---

## 0. 녹화 전 준비 (Pre-flight 체크리스트)

| # | 항목 | 방법 |
|---|---|---|
| 1 | 관리자 화면에 보여줄 실측 데이터 채우기 | `node tests/seed_firestore.js` (가상 학생 5명) |
| 2 | 결과지를 "준비된 상태"로 띄워두기 | **아래 [준비 A/B] 중 택1** |
| 3 | 브라우저 캐시 깔끔하게 | 시크릿 창(Ctrl+Shift+N) 권장 — 옛 PWA 캐시·이어서진행 모달 회피 |
| 4 | 화면 정리 | 북마크바 숨김(Ctrl+Shift+B), 알림 끄기, 확대 100%(Ctrl+0) |
| 5 | 모바일 느낌 내기 | DevTools(F12) → 디바이스 툴바(Ctrl+Shift+M) → iPhone 12 Pro |
| 6 | URL 미리 열어두기 | 학생 탭(`/`) + 관리자 탭(`/#/admin`) 2개 |

### 준비 A — 실제 검사 1회 완료 (가장 자연스러움)
녹화 **전에** 같은 탭에서 검사를 끝까지 1회 응답해 둔다. sessionStorage가 탭 유지 동안 남아
`#/result`가 즉시 뜬다. (검사 키보드 단축키 `1`~`5`로 빠르게 응답 — 90문항 약 1~2분)

### 준비 B — 결과지 미리 캡처본 사용 (라이브 부담 줄이기)
라이브로 90문항을 보여주기 부담되면, STEP3는 **문항 3~4개만 시연**하고 결과지는
[`assets/04_result_full.png`](assets/04_result_full.png)로 컷 전환(편집). 가장 안전한 방식.

> **권장 구성**: STEP1·2·3는 라이브, 결과지·수강계획서·관리자는 라이브 또는 캡처본 컷.
> 편집 컷이 1회 들어가도 3분 영상에서는 자연스럽다.

## 2. 자주 하는 실수 (NG 모음)

- ❌ STEP3에서 90문항을 라이브로 다 채우려다 시간 초과 → 3~4문항만, 결과는 컷
- ❌ 옛 PWA 캐시로 화면이 옛 버전 → **시크릿 창**으로 녹화

- ❌ 관리자 화면을 모바일 폭으로 → 사이드바가 햄버거로 접힘. **데스크탑 폭**으로
- ❌ 화면에 학번·이름·실명 노출 → 닉네임은 반드시 가명(`민서` 등)
