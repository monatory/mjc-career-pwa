# 시연(Demo) 스킬

MJC-CAT 학과추천 진단 PWA의 **시연 영상·발표 자료**를 다루는 스킬.
"시연 엠디"라는 호출의 실체가 이 스킬이다.

## 발동 시 먼저 할 일

1. **항상 [`docs/demo/README.md`](../../../docs/demo/README.md)를 먼저 읽는다.**
   그 안의 "파일 목록(매니페스트)" 표가 현재 시연 자료의 단일 진실 출처(SSOT)다.
2. 작업 대상이 명확하지 않으면, 매니페스트의 ⏳ 예정 항목 중 무엇을 진행할지 사용자에게 확인한다.

## 이 스킬이 다루는 폴더

```
docs/demo/
  README.md                  ← 매니페스트 (자료 목록·상태). 자료 추가 시 표에 한 줄 추가 필수
  notebooklm_demo_3min.md    ← NotebookLM Video Overview용 3분 소스 (7 SCENE)
  (이하 예정) assets/ · cue_sheet.md · demo_1min.md · demo_5min.md
```

스킬 자신의 정의는 `.Codex/skills/demo/SKILL.md`(이 파일).

## 작업 규칙

- **분리 원칙**: 이 스킬은 시연 산출물만 만든다. `data/`·`lib/`·`src/` 등 시스템 본체는
  건드리지 않는다. 본체 변경이 필요하면 일반 작업으로 전환하고 사용자에게 알린다.
- **매니페스트 동기화**: `docs/demo/`에 파일을 새로 추가/삭제하면 반드시
  `docs/demo/README.md`의 매니페스트 표를 같이 갱신한다.
- **사실 일관성**: 시연 자료의 수치·URL·기능 설명은 루트 `AGENTS.md`와 실제 코드를
  진실 기준으로 삼는다. 과장하거나 미구현 기능을 "완성"으로 적지 않는다.
- **언어**: 모든 시연 산출물은 한국어. 청중은 학내 의사결정자·학생·교수진 혼합.
- **NotebookLM 적합 형식**: 영상 자동 생성 소스는 SCENE 단위 + 화면 설명 + 내레이션
  구조를 유지한다(이미 `notebooklm_demo_3min.md`가 그 틀). 변형판(1분·5분)도 같은 틀.

## 자주 하는 작업

| 요청 예시 | 할 일 |
|---|---|
| "시연 엠디 열어줘 / 보여줘" | `docs/demo/README.md` + 해당 대본 파일을 읽고 요약·표시 |
| "3분 대본 고쳐줘" | `notebooklm_demo_3min.md`의 SCENE 수정, 매니페스트 상태 유지 |
| "1분 버전 만들어줘" | `demo_1min.md` 신규(같은 SCENE 틀, 축약), 매니페스트에 한 줄 추가 |
| "스크린샷 떠줘" | preview_* 도구로 STEP1·결과지·관리자 캡처 → `docs/demo/assets/` 저장, 매니페스트 갱신 |
| "큐시트 만들어줘" | `cue_sheet.md` 신규(클릭 순서·키 입력·타임라인) |

## 마무리

- 자료를 만든 뒤에는 빌드/테스트가 필요 없는 문서 작업이면 바로 커밋 가능
  (시스템 코드 무변경이므로 `npm run build` 회귀 대상 아님).
- 변경 요약을 사용자에게 보고하고, 필요 시 커밋·푸시 여부를 확인한다.
